## Notebooks as starting points

The code here represent useful notebook-type coding. Typically, these are workflows that require dynamic modfication as you proceed through a series of steps, and so static scripts (like the contents of the `scripts` directory) are not very effective. As such, these are more of 'notebooks' that are the equivalent of a lab notebook documenting a specific experiment, rather than a 'protocol' that is run consistently like the contents of the scripts folder. 

Note that these are NOT meant to be run directly, and likely need a fair bit of re-formatting to adjust the file naming conventions, paths, grouping of samples, and more. Typically the more efficient source would be copying these from a recent project with similar provenance and goals, not from here, as the notebooks here are examples to be kept in case of loss.

To recap:

**do not expect to apply these to your data as-is, they will need modification**

**these are potentially out of date and not the most efficient starting point**
